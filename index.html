Permission micro bloquée pour ce site

Le navigateur ne peut plus afficher la popup : tu dois réinitialiser l'autorisation dans les réglages du site, puis revenir ici.

Chrome / Edge
  1. Clique le cadenas (ou l'icône à gauche de l'URL).
  2. Ouvre Paramètres du site (ou Autorisations du site).
  3. Passe Microphone sur Autoriser, ou clique Réinitialiser les autorisations.
  4. Reviens sur cet onglet et clique le bouton ci-dessous.
Firefox
  1. Clique le cadenas dans la barre d'adresse.
  2. Dans Autorisations, mets Utiliser le microphone sur Autoriser.
  3. Ou clique Effacer les permissions puis recharge la page.
  4. Reviens ici et clique le bouton ci-dessous.
Safari
  1. Menu Safari → Réglages → Sites web → Microphone.
  2. Trouve ce site et passe-le sur Autoriser.
  3. Reviens sur cet onglet et clique le bouton ci-dessous.

Micro bloqué en HTTP (réseau local)

Les navigateurs refusent le micro hors HTTPS — sauf localhost, un contournement navigateur, ou un passage en HTTPS.

Chromium (Chrome, Edge, Brave…) : dans les paramètres du site, le micro est grisé et bloqué (« Bloquée pour protéger la confidentialité »). Le cadenas ne permet pas de l'autoriser : il faut localhost, HTTPS, ou le flag de lancement ci-dessous.

Chrome / Edge / Chromium — autoriser ton URL en HTTP
  1. Le cadenas ne fonctionne pas sur une IP en HTTP : le micro reste sur « Bloquer » et grisé.
  2. Sur cette machine, ouvre plutôt le lien localhost ci-dessous (recommandé).
  3. Ou ferme Chromium et relance-le avec le flag :
    --unsafely-treat-insecure-origin-as-secure=http://ton-ip/
  4. Ou sers le site en HTTPS (certificat auto-signé sur ton serveur web).
  5. Recharge cette page, puis lance la dictée depuis le Workspace.
Firefox — autoriser le micro en HTTP (LAN)
  1. Ouvre about:config dans la barre d'adresse, accepte l'avertissement.
  2. Passe media.devices.insecure.enabled à true (détection du micro).
  3. Passe aussi media.getusermedia.insecure.enabled à true (accès au micro — indispensable).
  4. Recharge cette page, puis lance la dictée depuis le Workspace.

Les deux réglages sont nécessaires : le premier détecte le micro, le second autorise son utilisation.

Sur cette machine, solution la plus simple — ouvre : localhost (Chromium autorise le micro sur localhost en HTTP)

Documentation

Dernière mise à jour : juillet 2026

Workspace

Colle ou dicte ton prompt dans la zone de saisie, active les prompts de contexte souhaités (tags rouges), puis lance le nettoyage. Le résultat apparaît à droite ; tu peux copier le prompt nettoyé ou relancer une passe.

  • Dictée vocale : option secondaire, 100 % locale. Le texte dicté reste modifiable avant nettoyage. Micro et moteur vocal se règlent dans Options → Dictée vocale.
  • Prompts de contexte : clique sur un tag pour l’activer (rouge = actif). L’ordre d’injection par rapport au prompt système se configure dans Prompts.
  • Historique : chaque nettoyage texte est enregistré localement. Les entrées peuvent être supprimées une par une ou vidées en bloc.
  • Nettoyage bloqué : active le prompt système ou au moins un prompt de contexte dans l’onglet Prompts.

Prompts

Prompt système (optionnel)

Quand il est actif, ses règles s’appliquent à chaque nettoyage. Quand il est inactif, seuls les prompts de contexte actifs sont envoyés au modèle.

Laissé vide, le prompt par défaut de PromptDeMerde est utilisé.

Prompts de contexte

Crée autant de contextes que nécessaire (un tag avec #, ex. #TonFormel, plus des instructions). Choisis si les contextes passent avant ou après le prompt système ; le schéma se met à jour selon ton choix.

La génération assistée (par titre ou par intention) utilise le fournisseur et le modèle configurés dans le Workspace.

Historiques LLM

Journal local des nettoyages texte (pdm_clean_history). Chaque entrée peut être supprimée individuellement via le bouton dédié.

Options

Intelligence artificielle

Choisis le fournisseur, l’URL du moteur LLM et le modèle. Teste la connexion avant usage. La réflexion du modèle n’est disponible que sur les modèles qui la supportent.

Thème

Le sélecteur de thème choisit l’apparence ; le bouton 🌙 dans le header bascule clair/sombre.

Dictée vocale

Moteurs 100 % locaux dans le navigateur :

  • Vosk Mini (~39 Mo) : léger, articulation nette, pas d’autocorrection.
  • Vosk Maxi (~41 Mo) : meilleur Vosk compatible navigateur.
  • Whisper (4 bits / q8) : autocorrection, GPU conseillé.
  • Parakeet (~409 Mo) : précis, GPU conseillé. Si le modèle manque : bash assets/stt/parakeet/restore-encoder.sh.

En HTTP sur une IP (hors localhost), Chromium bloque le micro : utilise localhost, HTTPS ou les contournements décrits dans Options quand le panneau s’affiche.

Sauvegarde & restauration

L’application ne stocke rien sur le serveur. Options, paramètres, historique et brouillon Workspace résident dans le navigateur (localStorage).

L’export JSON produit un fichier promptdemerde-config-v1.4.0.json de type pdm-config : sauvegarde complète (prompts, LLM, dictée, historique, thème, token API Ollama). L’import remplace intégralement les données locales après confirmation.

Format pdm-config — règles strictes

Le JSON est la base de données portable de l’application. Tu peux l’éditer à la main, mais l’import exige un fichier complet et valide :

  • Toutes les clés obligatoires doivent être présentes (17 clés pdm_* + version + type).
  • Aucune clé inventée à la racine (ex. pdm_foo → rejet).
  • Types et valeurs contrôlés : booléens natifs (true/false), enums (langue, thème, moteur STT…), URL Ollama valide, tags profils conformes.
  • Pas d’import partiel : les anciens formats system-prompt et prompts ne sont plus acceptés.
  • Rejet sans perte : si le fichier est invalide, ta session actuelle n’est pas effacée ; un message d’erreur détaillé s’affiche.

Politique de Confidentialité

Dernière mise à jour : 14 juin 2026

1. Résumé exécutif

PromptDeMerde.com ne conserve aucune donnée sur le serveur. Aucune base de données, aucun compte utilisateur. Vos options, paramètres, historique des nettoyages et brouillons sont enregistrés uniquement dans le stockage local de votre navigateur.

Le traitement des prompts s’effectue via un moteur LLM local, à configurer sur votre machine ou votre réseau. Un relais technique assure la connexion, sans persistance des contenus transités.

Pour sauvegarder, transférer ou restaurer l’ensemble de vos données d’usage (réglages, prompts, historique, configuration LLM, dictée vocale, token API), utilisez la fonction Export / Import dans Options → Sauvegarde & restauration. Seul un fichier JSON pdm-config complet et valide est accepté à l’import.

  • La dictée vocale est traitée dans le navigateur ; aucun envoi audio vers un service tiers
  • La sauvegarde durable de l’ensemble de ces données passe par l’export JSON (Options → Sauvegarde & restauration)

4. Cookies

PromptDeMerde.com n’utilise aucun cookie, ni cookie de session, ni cookie de tracking, ni cookie tiers.

5. Tracking et analytique

PromptDeMerde.com ne collecte aucune donnée analytique. Pas de Google Analytics, pas de Matomo, pas de pixel de tracking, pas de fingerprinting. Votre navigation est entièrement privée.

6. Services tiers

Le nettoyage des prompts utilise un moteur LLM sur votre machine ou votre réseau. La dictée vocale embarque des modèles locaux (Vosk, Whisper, Parakeet) sans appel à un service cloud pour l’audio.

  • Droit à la portabilité : utilisez Exporter ma configuration dans Options pour obtenir un fichier JSON de l’ensemble de vos données d’usage

8. Durée de conservation

Les données sont conservées dans votre navigateur jusqu’à ce que vous les supprimiez manuellement (bouton « Tout Effacer ») ou via les paramètres de votre navigateur.
