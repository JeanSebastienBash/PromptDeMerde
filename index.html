Profil JSON

Si le fichier assets/profiles/manifest.json est présent sur le serveur (local maintainer, pré-prod ou prod), un sélecteur Profil JSON apparaît dans le header. Chaque profil charge un pack complet : prompt système, prompts de contexte, moteur STT, thème, exemple workspace.

id (technique)LabelUsage
ads-generatorAdsGeneratorPetites annonces entre particuliers
speech2texteSpeech2TexteCorrection post-dictée (défaut)
video-meta-descriptorVideoMetaDescriptorMétadonnées vidéo YouTube

Changer de profil efface toutes les données locales puis recharge la config assemblée cété serveur (une seule confirmation). Le profil sélectionné est mémorisé dans pdm_active_profile — clé non incluse dans l’export JSON. Un import manuel via Options l’efface : le sélecteur retombe sur Speech2Texte au rechargement.

Sur un clone GitHub sans bundle profils, ce sélecteur n’apparaît pas.

Réglages communs à tous les profils : Vosk Maxi, CPU, contextes injectés après le prompt système.

Speech2Texte (speech2texte) — profil par défaut

Workspace

Colle ou dicte ton prompt dans la zone de saisie, active les prompts de contexte souhaités (tags rouges), puis lance le nettoyage. Le résultat apparaît à droite ; tu peux copier le prompt nettoyé ou relancer une passe.

  • Dictée vocale : option secondaire, 100 % locale. Le texte dicté reste modifiable avant nettoyage. Moteur vocal, micro et accélération CPU/GPU se règlent dans le bloc dictée du Workspace.
  • Prompts de contexte : clique sur un tag pour l’activer (rouge = actif). L’ordre d’injection par rapport au prompt système se configure dans Prompts.
  • Historique : chaque nettoyage texte est enregistré localement dans le panneau Historique LLM du Workspace. Les entrées peuvent être supprimées une par une ou vidées en bloc.
  • Nettoyage bloqué : active le prompt système ou au moins un prompt de contexte dans l’onglet Prompts.

Prompts

Prompt système (optionnel)

Quand il est actif, ses règles s’appliquent à chaque nettoyage. Quand il est inactif, seuls les prompts de contexte actifs sont envoyés au modèle.

Le textarea affiche le contenu de pdm_system_prompt s’il est renseigné, sinon le prompt Nettoyeur intégré. Après changement de profil JSON, le prompt importé remplit ce champ. L’onglet Prompts se resynchronise à chaque ouverture.

Prompts de contexte

Crée autant de contextes que nécessaire (un tag avec #, ex. #TonFormel, plus des instructions). Choisis si les contextes passent avant ou après le prompt système ; le schéma se met à jour selon ton choix.

Réorganise l’ordre des contextes par glisser-déposer (poignée ⋮⋮) ou boutons ↑↓ : cet ordre s’applique aux tags du Workspace et à l’injection LLM (contextes actifs uniquement).

La génération assistée (par titre ou par intention) utilise le fournisseur configuré dans Options et le modèle choisi dans le Workspace.

Sur grand écran, l’onglet s’affiche en deux colonnes : prompt système à gauche, prompts de contexte à droite.

Options

Intelligence artificielle

Choisis le fournisseur et l’URL du moteur LLM, puis teste la connexion. Le modèle et la réflexion se configurent dans le Workspace (en-tête du panneau résultat). Si aucun modèle n’est défini (premier lancement ou profil sans modèle), le premier modèle Ollama disponible est sélectionné automatiquement — la réflexion est alors désactivée.

La réflexion du modèle n’est disponible que sur les modèles qui la supportent. Un plafond de caractères limite la longueur affichée (pdm_llm_thinking_max_chars, 0 = illimité).

Thème

23 thèmes disponibles (12 clairs : Cassé, Jour, Orange clair… — 11 sombres : Sombre, Orange, Océan…). Le bouton 🌙 dans le header fait défiler tous les thèmes ; la grille complète est dans Options.

Dictée vocale

Moteurs 100 % locaux dans le navigateur (réglages dans le bloc dictée du Workspace) :

  • Vosk Mini (~39 Mo) : léger, articulation nette, pas d’autocorrection.
  • Vosk Maxi (~41 Mo) : meilleur Vosk ; valeur par défaut si aucun moteur n’est encore enregistré.
  • Whisper Mini / Whisper Maxi : autocorrection, GPU conseillé.
  • Parakeet (~409 Mo) : précis, GPU conseillé. Si le modèle manque : bash assets/stt/parakeet/restore-encoder.sh.

Le choix CPU / GPU (Whisper, Parakeet) n’apparaît que si le navigateur détecte un GPU WebGPU avec ≥ 1 Go VRAM.

En HTTP sur une IP (hors localhost), Chromium bloque le micro : utilise localhost ou HTTPS.

Sauvegarde & restauration

L’application ne stocke rien sur le serveur. Options, paramètres, historique et brouillon Workspace résident dans le navigateur (localStorage).

L’export JSON produit un fichier promptdemerde-config-v1.11.0.json de type pdm-config : sauvegarde complète (prompts, LLM, dictée, historique, thème, token API Ollama, personnalisation Workspace pdm_workspace_ui, blobs dictée optionnels pdm_audio_blobs). Si un token Ollama est configuré, une confirmation est demandée avant export (token en clair dans le fichier).

L’import valide d’abord le fichier, demande confirmation, puis remplace intégralement les données locales. Il efface aussi pdm_active_profile (sélecteur Profil JSON → Speech2Texte au reload).

Format pdm-config — règles strictes

Le JSON est la base de données portable de l’application. Tu peux l’éditer à la main, mais l’import exige un fichier complet et valide :

  • Toutes les clés obligatoires doivent être présentes (18 clés pdm_* + version + type).
  • Aucune clé inventée à la racine (ex. pdm_foo → rejet).
  • Types et valeurs contrôlés : booléens natifs (true/false), enums (langue, thème, moteur STT…), URL Ollama valide, tags profils conformes.
  • Pas d’import partiel : les anciens formats system-prompt et prompts ne sont plus acceptés.
  • Rejet sans perte : si le fichier est invalide, ta session actuelle n’est pas effacée ; un message d’erreur détaillé s’affiche.
  • Noms de fichier réservés (bundle profils actif) : l’import manuel est refusé si le nom du fichier (sans extension) correspond à un profil officiel. La comparaison ignore la casse et tout caractère non alphanumérique. Exemples refusés : Speech2Texte.json, AdsGenerator.json. Le sélecteur Profil JSON du header n’est pas concerné.
  • Ordre import : validation du JSON → confirmation → effacement → application → rechargement. Fichier invalide : session intacte.

Politique de Confidentialité

Dernière mise à jour : 9 juillet 2026

1. Résumé exécutif

PromptDeMerde.com ne conserve aucune donnée sur le serveur web après vos requêtes. Aucune base de données, aucun compte utilisateur, aucune session PHP. Vos options, paramètres, historique des nettoyages et brouillons sont enregistrés uniquement dans le stockage local de votre navigateur.

Le traitement des prompts s’effectue via un moteur LLM local, à configurer sur votre machine ou votre réseau. Un relais technique assure la connexion, sans persistance des contenus transités.

Pour sauvegarder, transférer ou restaurer l’ensemble de vos données d’usage (réglages, prompts, historique, configuration LLM, dictée vocale, token API), utilisez la fonction Export / Import dans Options → Sauvegarde & restauration. Seul un fichier JSON pdm-config complet et valide est accepté à l’import.

3. Flux de données

Lorsque vous utilisez le service :

  • Vos prompts et contextes sont transmis à votre moteur LLM local — aucune conservation côté serveur
  • La réponse du modèle revient dans votre navigateur et peut être enregistrée localement (historique)
  • Options, historique et brouillons sont enregistrés localement dans le navigateur — non stockés sur le serveur web
  • La dictée vocale et l’import fichier audio sont traités dans le navigateur ; aucun envoi audio vers le serveur
  • La sauvegarde durable de l’ensemble de ces données passe par l’export JSON (Options → Sauvegarde & restauration)

4. Garanties et limites

Garantie produit : aucune base de données, aucune session PHP, aucune télémétrie, aucune écriture applicative des prompts sur disque. Code open source auditable. L’auto-hébergement permet de contrôler entièrement l’infrastructure.

Transit LLM : lors d’un nettoyage, votre texte, le prompt système et les contextes actifs traversent le proxy en mémoire (POST api/chat) pour joindre Ollama — puis la requête se termine. Rien n’est archivé côté serveur par l’application.

Limite infrastructure : un opérateur d’hébergement pourrait journaliser le trafic HTTP s’il le configurait — ce n’est pas une fonctionnalité du produit.

5. Cookies

PromptDeMerde.com n’utilise aucun cookie, ni cookie de session, ni cookie de tracking, ni cookie tiers.

6. Tracking et analytique

PromptDeMerde.com ne collecte aucune donnée analytique. Pas de Google Analytics, pas de Matomo, pas de pixel de tracking, pas de fingerprinting. Votre navigation est entièrement privée.

7. Services tiers

Le nettoyage des prompts utilise un moteur LLM sur votre machine ou votre réseau. La dictée vocale embarque des modèles locaux (Vosk, Whisper, Parakeet) sans appel à un service cloud pour l’audio.

8. Vos droits (RGPD)

Conformément au Règlement Général sur la Protection des Données (RGPD), vous disposez des droits suivants :

  • Droit d’accès : vos données étant stockées localement, vous y avez un accès direct via les outils de développement de votre navigateur (F12 → Application → Local Storage)
  • Droit de rectification : vous pouvez modifier vos données directement dans l’interface (Options, Prompts)
  • Droit à l’effacement : utilisez le bouton « Tout Effacer » dans Options, ou effacez les données du site via votre navigateur
  • Droit à la portabilité : utilisez Exporter ma configuration dans Options pour obtenir un fichier JSON de l’ensemble de vos données d’usage

9. Durée de conservation

Les données sont conservées dans votre navigateur jusqu’à ce que vous les supprimiez manuellement (bouton « Tout Effacer ») ou via les paramètres de votre navigateur. Rien n’est conservé sur le serveur web après vos requêtes.
